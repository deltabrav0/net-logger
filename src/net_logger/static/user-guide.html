Net Logger User Guide

A quick guide for running nets, checking in stations, updating FCC data, and exporting records.

Opening Net Logger

Start the server, then open Net Logger in a browser. The default local address is:

http://127.0.0.1:8088

If the server was started on all LAN interfaces, another device on the same network can open the app by using the host computer's LAN address and configured port.

Basic use

  1. Enter a net name, frequency, and Net Control callsign.
  2. Press Start Net.
  3. Net Control is automatically added to Checked In.
  4. Check in stations with the single station lookup box, a card's Check in button, or drag-and-drop.
  5. Expand a checked-in card to edit traffic and notes.
  6. Press Stop Net when the net is finished.
  7. Open Saved Nets / Metrics for CSV export or metrics as needed.

Starting a net

At the top of the page, enter the net name, frequency, and Net Control callsign. After you have saved at least one net, Net Logger pre-fills Net name and Frequency with the most recent saved values. You can still type over either field at any time.

Press Start Net. If Net Control is not already known, Net Logger attempts a local FCC lookup when FCC data is configured. Net Control is then automatically added to the Checked In column for the active session.

Searching, adding, and checking in stations

Enter a callsign or operator name in the single station lookup box labeled Search or add station. As you type, Net Logger shows reusable known-station suggestions with saved name, location, and grid details. If the typed callsign exactly matches a known station, the helper text confirms the saved station details before you press Enter.

Net Logger searches known stations first. If no known callsign matches, it then searches the local FCC database. If FCC data is found, the station is saved with FCC details; otherwise a callsign-only station is saved.

When a net is open, a known station selected from this box is reused and checked into the active net automatically. When no net is open, the station list is filtered as you type and new stations are saved for later use.

You can also check in a known station by clicking Check in on the station card or dragging the station card from Known Stations to Checked In.

If a manually added station is later available in FCC records, click Update details on its Known Stations card. Net Logger looks up that station's callsign in the local FCC database and updates its name, location, grid, latitude, and longitude. Use Delete only when a station record is wrong or unwanted; deleting a station also removes saved check-ins for that station.

Editing traffic and notes

Expand a checked-in station card to edit whether the station has traffic, traffic details, and notes.

FCC database age and updates

The toolbar shows the local FCC database age, for example FCC database: 3 days old. If local FCC data is missing, the indicator shows that the database is unavailable.

Press Update FCC Database to download the current FCC amateur-license file, extract the local lookup files, and rebuild the callsign index. The update requires internet access and can take a few minutes. While it is running, the toolbar shows FCC database: updating…. When it finishes successfully, the age should change to updated today.

From a Docker host, check status with curl http://127.0.0.1:8088/api/fcc/status. A healthy updated database reports available: true, age_days: 0, and present EN.dat / EN.idx files. You can also run docker compose exec net-logger ls -lh /fcc/data/EN.dat /fcc/data/EN.idx /fcc/data/HD.dat.

Last Heard timestamps

Known station cards display Last Heard information when available. This helps Net Control quickly see which stations have checked in recently.

Stopping or canceling a net

Press Stop Net when the net is finished. The session and check-ins are saved in the SQLite database.

Press Cancel Net when the net was started by mistake or should be discarded. Canceling deletes that net session and its check-ins so it does not appear in Saved Nets, exports, or Metrics. Known station records remain available.

CSV export

Use CSV export to download saved net records for spreadsheet review, archival records, or external reporting. In the web app, open Saved Nets / Metrics from the main page header to reach saved sessions and export links.

Metrics

Open Saved Nets / Metrics from the main page header to view metrics. The Metrics view summarizes check-in activity by net and by week, month, or year. Use the Net name dropdown to show one saved net name, or choose All nets to compare every saved net.

WordPress export

If the Net & Meeting Attendance WordPress plugin is configured, open Saved Nets / Metrics and press Send to WordPress on a saved net.

If the WordPress endpoint, username, or Application Password are missing, Net Logger opens a setup form automatically. Enter the settings, then choose Test Only to check the connection without saving, or Test and Save to check the connection and write the settings to config.ini.

The WordPress settings live in Net Logger's config.ini file under [wordpress]. A configuration file is just a small text file. You can still edit it by hand, save it, and restart Net Logger.

[wordpress]
endpoint = https://example.org/wp-json/net-attendance/v1/net-logger/sessions
username = your-wordpress-username
application_password = paste-your-wordpress-application-password-here
timeout = 20

Net Logger records a successful push locally and disables the button for that saved net afterward.

Backing up data

Stop the server, then copy the SQLite database file. On macOS with default installed paths:

cp "$HOME/Library/Application Support/Net Logger/net_logger.sqlite3" ./net_logger_backup.sqlite3
